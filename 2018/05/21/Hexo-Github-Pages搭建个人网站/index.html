Hexo+Github Pages搭建个人网站

2018-05-21

Hexo+Github Pages搭建个人网站

2018-05-21

概述

　　前几天在网上看到一个博客主题比较好看，就自己动手搭建了一个，第一次搭建完成大概用了一下午，后来由于更换主题和域名的原因花了两天时间，总的整好花了大概5天时间。如果你要求不高，那么最多一天也能搭建起来。
　　为什么选择hexo和GitHub呢？首先它能快速生成静态网页，简单美观，其次GitHub提供了300m的免费空间。
　　下面正式开始喽！

搭建博客的基本步骤

–安装git
–安装node.js
–安装hexo
–注册GitHub账号

一.安装git和node.js

　　安装工具主要是为了又一个运行环境，更好的让我们使用hexo。
　　安装git和node.js非常简单，博主在这里就不在赘述了，注意自己的电脑是Windows还是Mac下载对应的安装包，安装就可以了。
　　Git安装包下载
　　node.js安装包下载
安装完成后我们继续，如果实在不会安装可以找我，有时间一定回。

小白请注意！高手自动忽略一下内容
git入门教程

二.安装hexo

　　现在我假设你已经正确安装了以上软件，打开git命令板(windows右键可看到),进入文件夹，
例如我在桌面创建的文件夹：

下载安装hexo,执行下面语句：

$ npm install -g hexo-cli

执行下面语句，初始化博客(执行时确保你所在的文件夹路径正确)：

$ hexo init <floder>
//folder为你的博客存放的文件夹

打开文件加看一下你的生成目录，例如我的：

至此你的博客已经初步形成可以在本地运行
测试博客　
1.生成一片文章
$ hexo new “测试1”
此时会在你的本地目录source/_post生成了一个”测试1.md”的文件，编辑可修改。
2.生成一个可执行文件

$ hexo g

3.启动本地服务器

$ hexo s

4.默认会在4000端口启动，我们在浏览器输入http://localhost:4000即可查看，有没有小激动呢！

只有我们自己能看到这怎么行呢？既然玩了就要搞得完美，我么继续，通过下面的操作你的网站别人就能访问了，期待~

三.注册GitHub

　打开网站git注册就行，这里不多讲。
　创建一个仓库，在网站的右上角可以看到

点击new repository创建一个仓库

注意仓库的名字要填：你的用户名.github.io
description随便填就OK啦！
public要选中，底下的initilize this repository with a README选中。
点击绿色按钮就可以了
由于我们初次创建所以要填写SSH值
详细步骤点击链接

四.将博客部署到GitHub Pages上

找到你的博客根目录，有一个_config.yml文件,用文本编辑器打开，最好不用Windows自带的记事本，具体原因不做解释。找到deploy字段配置如下：

deploy:
type: git
repo: git@github.com:你的账号名/你的仓库名(例如我的：git@github.com:angeraly/angeraly.github.io)
branch: master
注意安装一个扩展
$ npm install hexo-deployer-git --save
然后执行
hexo d

非常简单就成功了，当然由于第一次部署，或者莫名的原因不成功，那么我们可以直接用git上传
下面简单介绍如果上面已经成功请不要继续看

$ cd 你的文件夹
$ hexo g
$ cd public
$ git init
$ git add .
$ git commit -m "message"
$ git remote add origin git@github.com:你的账号/你的仓库
$ git push -u origin master

好哒大功告成，直接浏览器输入你的resporitory，接下来就去看一下你的博客吧！

总结

搭建博客的方法很多，有兴趣的可以看一下WordPress搭建博客，后期管理比较方便，以后有时间搞了会出教程。关于域名绑定的问题可以自行百度，以后有时间更新，一下子码完这么多字，累~~~~
